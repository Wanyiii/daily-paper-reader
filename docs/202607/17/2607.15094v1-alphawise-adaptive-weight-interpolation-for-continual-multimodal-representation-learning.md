## 摘要
多模态模型如CLIP学习一个共享的嵌入空间用于跨模态检索，但持续适应依次到达的数据可能会破坏早期阶段获得的跨模态对齐。传统的持续学习方法返回单个检查点，这使每个检索方向陷入相同的稳定性-可塑性权衡。我们提出AlphaWiSE，一种事后权重空间插值方法，它组合两个冻结的源检查点。对于由其检查点键标识的每个对齐参数张量，AlphaWiSE拟合一个由所有张量条目共享的标量插值系数。这些系数在一个较小的样本记忆上拟合，并用于具体化一个插值检查点。部署的模型具有与任一源检查点相同的架构和参数数量，不需要额外的推理时间。在音频-图像-文本检索上的大量实验显示，在多个检索方向和评估指标上相较于强持续学习基线有一致的改进。

## Abstract
Multimodal models such as CLIP learn a shared embedding space for cross-modal retrieval, but continual adaptation to sequentially arriving data can disrupt the cross-modal alignment acquired from earlier phases. Conventional continual-learning methods return a single checkpoint, which commits every retrieval direction to the same stability-plasticity trade-off. We propose AlphaWiSE, a post-hoc weight-space interpolation method that composes two frozen source checkpoints. For each aligned parameter tensor identified by its checkpoint key, AlphaWiSE fits one scalar interpolation coefficient shared by all tensor entries. The coefficients are fitted on a smaller exemplar memory and used to materialize one interpolated checkpoint. The deployed model has the same architecture and parameter count as either source checkpoint, which does not require additional inference time. Extensive experiments on audio-image-text retrieval show consistent improvements over strong continual-learning baselines across multiple retrieval directions and evaluation metrics.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：多模态模型（如 CLIP、AudioCLIP）在持续学习新数据时，会破坏早期学到的跨模态对齐关系，导致检索性能下降。传统的持续学习方法（如 EWC、LwF、iCaRL）虽然能缓解灾难性遗忘，但最终只返回一个单一检查点，迫使所有检索方向（如音频→文本、图像→音频）共享同一稳定性-可塑性权衡。
- **整体含义**：不同持续学习策略产生的检查点保留了各自不同的跨模态几何特性，没有一个单独检查点能同时对所有检索方向最优。因此，作者提出**事后（post-hoc）权重空间插值**，将两个冻结的持续学习检查点融合，从而在保留架构和推理成本不变的前提下，获得更好的多模态检索表现。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- 将两个互补的持续学习检查点（一个偏向可塑性，如标准顺序微调；另一个偏向稳定性，如 EWC/LwF/iCaRL）通过**张量级（tensor-level）的标量插值系数**进行加权融合。系数在小型样本记忆上优化，两个源检查点全程冻结。

### 关键技术细节
- **张量级插值**：每个命名参数张量 $p$ 对应一个标量系数 $\alpha_p = \sigma(\beta_p)$，$\beta_p$ 为可学习 logit，$\sigma$ 为 sigmoid 函数。融合后参数 $\tilde{\theta}_p = \alpha_p \theta_p^{\text{un}} + (1-\alpha_p)\theta_p^{\text{reg}}$。
- **系数初始化**：所有 $\beta_p=0$，即 $\alpha_p=0.5$，起始为等比例混合。
- **优化目标**：在样本记忆 $\mathcal{M}$ 上最小化跨模态检索损失 $\mathcal{L}_{\text{ret}}$，该损失是多个检索方向（如音频→文本、图像→音频、图像→文本）InfoNCE 损失的平均。
- **算法流程**（Algorithm 1）：
  1. 初始化 $\beta_p=0$。
  2. 循环 $T$ 步：从记忆 $\mathcal{M}$ 采样批次，计算 $\alpha_p$ 和融合参数 $\tilde{\theta}$，计算损失，更新 $\beta$。
  3. 最终根据学习到的 $\alpha^\star$ 物化融合检查点 $\tilde{\theta}^\star$。
- **梯度解释**：$\partial \mathcal{L}_{\text{ret}}/\partial \beta_p = \alpha_p(1-\alpha_p) \langle \nabla_{\tilde{\theta}_p} \mathcal{L}_{\text{ret}}, \theta_p^{\text{un}} - \theta_p^{\text{reg}} \rangle$，即根据两个检查点差异方向上的局部导数更新系数。

### 关键优势
- 融合后模型参数数量、架构、推理成本与任一源模型相同，无需额外适配器或路由机制。

## 6. 主要结论与发现
- AlphaWiSE 在所有检索方向和指标上一致优于单个持续学习基线及 WiSE-FT 全局系数插值。最佳配对取决于检索方向和指标：Normal+EWC 在音频→文本、图像→音频上最强；Normal+iCaRL 在图像→文本上表现最佳。
- **跨目标效应**：联合优化所有检索方向目标得到的系数对音频相关检索最优；仅优化 A→T 方向可提升 I→T，但弱化 I→A。说明不同目标偏好不同的参数混合。
- **插值系数分析**：参数**类型**（注意力权重、MLP/卷积权重、归一化尺度/偏置）是影响系数的主要因素，而非层深度。归一化参数吸收了大部分“可塑性更新”，权重矩阵则更倾向于保留稳定模型。
- **定性可视化**：AlphaWiSE 融合后的嵌入空间具有更紧凑的类内聚类和更好的跨模态对齐。

## 7. 优点
1. **后验融合，无需重新训练**：直接利用两个现成的持续学习检查点，通过轻量级系数优化即可获得更好模型。
2. **零额外推理成本**：最终模型与源模型架构相同，参数数相同，计算开销不变。
3. **张量级插值**：比全局系数（如 WiSE-FT）更具表达力，同时保持低维优化空间（仅 499 个系数对 AudioCLIP ViT-B/32）。
4. **不需要适配器或路由**：方法简洁，易于集成到现有工作流。
5. **实验设计全面**：包含主结果、消融、可视化、系数分析，验证了插值策略的有效性。

## 8. 不足与局限
1. **依赖两个特定源检查点**：方法需要一个“可塑性”检查点和一个“稳定性”检查点，且要求它们架构完全兼容（同一主干的不同训练轨迹），限制了灵活性。
2. **仅在 AudioSet 上验证**：缺少在更多样化的多模态持续学习基准（如 MSR-VTT、VQA 等）上的验证，泛化性存疑。
3. **记忆规模固定**：仅测试了 840 样本的记忆，未探索不同记忆大小对融合效果的影响，也未与无记忆的场景比较。
4. **需额外优化步骤**：虽然推理无额外成本，但系数优化需要额外的一轮训练（在记忆上），增加了后处理时间。
5. **未分析参数数量敏感性**：对于更大规模模型（如 ViT-L/14），张量数量可能显著增加，系数优化是否仍然高效有待验证。
6. **未讨论失败案例**：某些配对（如 Normal+LwF 在 I→A 上不如 EWC 版本）的原因缺乏深入分析。
