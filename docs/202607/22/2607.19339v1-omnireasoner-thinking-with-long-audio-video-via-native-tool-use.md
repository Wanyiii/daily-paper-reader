## 摘要
长音频视频推理对于全模态大语言模型来说是困难的，因为决定性证据通常稀疏、跨模态且成本高昂以至于无法以统一高保真输入来保留。我们提出OmniReasoner，一个用于长音频视频推理的工具使用后训练框架：全模态大语言模型通过监督微调和强化学习，学会在回答前决定是否以及在哪里调用放大工具。OmniReasoner首先构建整个流的低成本全局预览，然后在需要时，调用具有请求时间间隔的放大工具，以在回答前进行更高保真度的视觉和音频检查。由于模型在调用前后观察到不同的采样粒度——稀疏的全局预览和更密集的局部片段——我们引入了TimeAnchor，它使工具的时间参数在这些粒度之间保持有效和往返一致性，而不是绑定到特定采样率的帧索引。为了使这种工具使用行为可训练而不需要昂贵的人工间隔标注，我们构建了一个时间增强数据引擎，通过视频编辑和组合来合成工具使用的后训练轨迹。跨全模态和视频基准的实验表明，OmniReasoner既提高了答案准确性，又提高了时间定位能力，同时将高保真计算集中在信息区域。代码可在https://github.com/RockyChen0205/OmniReasoner获取。

## Abstract
Long audio-video reasoning is difficult for omnimodal LLMs because the decisive evidence is often sparse, cross-modal, and too expensive to preserve with uniformly high-fidelity inputs. We introduce OmniReasoner, a tool-use post-training framework for Thinking with Long Audio-Video: omni-modal LLMs learn, via supervised fine-tuning and reinforcement learning, to decide whether and where to call a zoom-in tool before answering. OmniReasoner first builds a low-cost global preview of the full stream and then, when needed, calls the zoom-in tool with a requested temporal interval for higher-fidelity visual and audio inspection before answering. Because the model observes different sampling granularities before and after this call -- a sparse global preview and a denser local clip -- we introduce TimeAnchor, which keeps the tool's temporal argument valid and round-trip-consistent across these granularities, rather than tied to frame indices from a particular sampling rate. To make this tool-use behavior trainable without expensive manual interval annotation, we build a Temporal Augmented Data Engine that synthesizes tool-use post-training trajectories by video editing and composition. Experiments across omnimodal and video benchmarks show that OmniReasoner improves both answer accuracy and temporal grounding while concentrating high-fidelity computation on informative regions. Code is available at https://github.com/RockyChen0205/OmniReasoner.

# OmniReasoner 论文总结

## 1. 核心问题与整体含义（研究动机和背景）

长音频视频推理面临一个根本性矛盾：决定性证据通常稀疏、跨模态（可能是短暂的视觉动作、语音片段、背景声音，或两者的时间巧合），且均匀高保真处理整个流在计算上不可行（密集帧和连续音频消耗大量上下文）。现有方法大多针对视觉证据设计，缺乏联合决定“看哪里、听哪里、何时两者交互是关键”的能力。OmniReasoner旨在通过可学习的工具使用行为，让全模态大语言模型学会在回答前决定是否以及在哪里调用放大（zoom-in）工具，以获取更高保真度的局部证据。

## 2. 方法论

### 核心思想
- **两阶段推理**：先构建低成本的全局预览（低帧率视频+完整音频），模型基于此决定是直接回答还是调用zoom-in工具（指定时间间隔）。若调用工具，则获得对应间隔的高保真音视频片段，结合全局上下文进行最终回答。
- **TimeAnchor机制**：为解决跨采样粒度（全局稀疏预览 vs 局部密集片段）的时间参数一致性问题，在Qwen-Omni的时间交错编码基础上，为每个2秒块添加纯文本绝对时间标记（如<0 seconds>），使工具调用的时间参数（秒为单位）在不同采样率下保持有效和往返一致。
- **时序增强数据引擎**：通过视频编辑和组合自动合成工具使用训练轨迹，包括多片段组合（将多个语义独立片段拼接成长视频，每段对应问题）和异常插入（替换音视频流的部分内容，标注异常类型和时间间隔），提供答案、间隔和工具使用行为的监督信号。

### 技术流程
1. 输入长音视频x和问题q。
2. 模型生成全局观测g = Φ_global(x)。
3. 策略πθ决定动作a1 ∈ {answer(y), zoom([s,e])}。
4. 若调用zoom，环境返回局部观测z_{s:e} = Φ_local(x,[s,e])。
5. 最终回答y ∼ πθ(·|g,q,z_{s:e})。
6. 使用GRPO进行强化学习优化，奖励函数R = R_acc + R_fmt（答案准确率+格式正确性）。

## 6. 主要结论与发现

1. **工具使用后训练显著提升长音视频推理**：OmniReasoner在OmniVideoBench（+5.5）、LVOmniBench（+3.4）、VideoHolmes（+15.6）等基准上均超过基线。

## 7. 优点

- **方法论创新**：将工具使用行为作为可学习的目标，而不是固定预处理规则，赋予模型自适应选择何时何地聚焦高保真计算的能力。
- **时间跨粒度一致性方案**：TimeAnchor设计简洁有效，无需新特殊token或架构改动，仅通过文本标记解决关键对齐问题。
- **数据引擎自动化**：通过视频编辑组合自动生成带间隔监督的训练轨迹，大幅降低人工标注成本，具有良好的可扩展性。
- **实验完备性**：消融实验设计细致，能清晰分离各组件贡献；行为分析增强了方法可解释性。
- **开源代码**：提供完整代码和数据集构造细节，可复现性强。

## 8. 不足与局限

1. **工具范围单一**：仅实现zoom-in工具，未探索搜索、代码执行、数据库查询等其他工具类型，限制了处理需要外部知识或符号推理的任务。
2. **基础设施限制**：
   - 多轮agentic RL的支持不成熟（自行搭建框架），无法扩展到更复杂的多步工具交互。
   - 模型使用Qwen2.5-Omni-7B（较小，32K上下文窗口），未预训练工具使用数据，导致直接RL效果差。
   - 未尝试更新更大的模型（如Qwen-Omni-3）因计算资源限制。
3. **数据合成偏差风险**：使用Gemini-3作为教师模型合成轨迹，可能引入教师模型的偏差或错误；虽然经过过滤，但幻觉和语义不一致仍可能存在。
4. **评估局限性**：
   - 主要依赖自动评估指标（准确率、IoU），未进行用户研究或主观质量评估。
   - 部分基准（如Daily-Omni、WorldSense）以短中视频为主，放大工具优势不突出。
   - 缺乏对音频模态单独破坏的消融（仅提到去除音频输入），未深入分析音频与视觉的协作程度。
5. **应用限制**：高保真计算集中在信息区域虽好，但若模型错误判断证据区域（如假阳性调用），可能浪费计算并引入错误证据。风险未量化分析。
