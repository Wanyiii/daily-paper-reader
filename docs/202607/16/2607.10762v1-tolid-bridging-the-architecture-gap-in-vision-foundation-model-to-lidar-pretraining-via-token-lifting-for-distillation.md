## 摘要
从视觉基础模型（VFM）到LiDAR骨干网络的跨模态蒸馏最近作为一种自监督预训练策略出现，减少了3D场景理解中对密集点级标注的依赖。然而，现有的蒸馏流程通常将VFM视为冻结的特征源，并训练异构的3D骨干网络以匹配固定的图像嵌入，迫使学生网络同时弥合模态差距和密集ViT令牌表示与稀疏3D编码器之间的跨架构差距。我们提出TOLiD，一种用于LiDAR表示学习的自监督预训练方法，通过将LiDAR骨干网络与从冻结VFM教师初始化的学生视觉Transformer（ViT）耦合，并在兼容的块-令牌表示上施加监督来解决这一差距。TOLiD使用视锥池化后接视锥注意力将每个图像块视锥内的点特征集转换为令牌，并通过可见性掩码执行令牌级蒸馏。对于仅LiDAR部署，我们使用掩码双线性采样将令牌特征提升回逐点表示，以避免LiDAR点有限的块。我们在五个异构LiDAR数据集和四个跨传感器适应对上广泛评估TOLiD，展示了使用冻结骨干网络和轻量级头部的改进转移性能。

## Abstract
Cross-modal distillation from Vision Foundation Models (VFMs) to LiDAR backbones has recently emerged as a self-supervised pretraining strategy that reduces reliance on dense point-wise annotation for 3D scene understanding. However, existing distillation pipelines typically treat the VFM as a frozen feature source and train a heterogeneous 3D backbone to match fixed image embeddings, forcing the student to bridge both the modality gap and the cross-architecture gap between dense ViT token representations and sparse 3D encoders. We propose TOLiD, a self-supervised pretraining method for LiDAR representation learning that addresses this gap by coupling a LiDAR backbone with a student Vision Transformer (ViT) initialized from a frozen VFM teacher and applying supervision over compatible patch-token representations. TOLiD converts the set of point features within each image patch frustum into a token using Frustum Pooling followed by Frustum Attention, and performs token-level distillation with visibility masking. For LiDAR-only deployment, we lift token features back to per-point representations using masked bilinear sampling to avoid patches that have limited LiDAR points. We extensively evaluate TOLiD on five heterogeneous LiDAR datasets and four cross-sensor adaptation pairs, demonstrating improved transfer with frozen backbones and lightweight heads.

- **核心问题**：如何弥合VFM（如DINOv2）与LiDAR骨干之间的架构鸿沟，使蒸馏过程更高效、语义更丰富。
- **整体含义**：提出TOLiD框架，通过**将LiDAR骨干与学生ViT耦合**，在兼容的patch token空间上施加蒸馏监督，从而缩小架构差异，实现更优的跨模态、跨传感器自监督预训练。

### 2. 论文提出的方法论
- **核心思想**：不直接对异构3D骨干进行蒸馏，而是引入一个**学生ViT**（初始化自冻结VFM教师权重），并与LiDAR骨干并行工作。LiDAR点特征通过**Frustum Pooling（FP）和Frustum Attention（FA）** 转换为与图像patch对应的token，然后与教师ViT输出的相同patch token进行对齐。这使学生ViT保留教师的推理能力，而LiDAR骨干则学习输出与token兼容的点特征。
- **关键技术细节**：
  - **Frustum Pooling（FP）**：在ViT输入阶段（j=0），将每个图像patch对应的3D视锥内的LiDAR点特征线性投影到ViT嵌入空间后取平均，形成patch token。无点的patch使用可学习的空token。
  - **Frustum Attention（FA）**：在更深层的ViT块（j>0）中，用交叉注意力机制让当前patch token作为query，视锥内点特征作为key/value，通过softmax加权聚合，使token可选择性关注其视锥内的几何信息。
  - **多阶段注入**：在J个ViT块处，LiDAR骨干前进对应的层数后，提取新的点特征，通过FP或FA注入学生ViT。
  - **可见性掩码蒸馏**：只在有LiDAR点投影的patch上计算局部patch token的蒸馏损失，避免对无几何支撑的区域强制对齐。损失函数为余弦相似度与smooth-L1的组合。
  - **掩码双线性采样（MBS）**：在微调阶段，将ViT token提升回逐点特征时，仅使用有有效token的四个邻域进行加权插值，并重新归一化，避免空patch引入噪声。
- **公式与流程**（文字说明）：
  - 点投影至图像：\( u_i = \pi(x_i) \)，定义可见点集。
  - FP：\( z_t = \frac{1}{|N(t)|} \sum_{i \in N(t)} W_p f_i \)。
  - FA：\( \Delta z_t = \sum_i \text{softmax}(Q_t^T K_i / \sqrt{d}) V_i \)，残差更新 \( z_t \leftarrow z_t + \text{LN}(\Delta z_t) \)。
  - 蒸馏损失 \( L = (L_{cls} + L_{patch})/2 \)，其中 \( L_{patch} \) 通过可见性掩码 \( M^v \) 加权。

### 6. 论文的主要结论与发现
- TOLiD通过**弥合架构差距**（使用学生ViT而非异构3D网络），显著提升了冻结骨干的迁移能力。
- 在LP、标签高效、域适应、鲁棒性上全面超越SOTA，尤其当预训练数据增加时，性能差距扩大（从+2.0% mIoU到+4.2% mIoU），证明其更好地利用数据多样性。

### 7. 优点
- **方法亮点**：
  1. **架构对齐**：首次将LiDAR骨干与学生ViT联合，使蒸馏在兼容token空间进行，避免了跨架构映射损失。
  2. **视锥注意力（FA）**：能够根据当前token语义状态动态选择重要点，比简单池化更具表现力。
  3. **掩码双线性采样**：简洁有效，防止空patch污染点特征。
  4. **可见性掩码蒸馏**：几何上合理的监督方式，只对存在LiDAR点的patch施加约束。
- **实验亮点**：
  1. 多数据集、多协议全面评估，验证泛化性。
  2. 跨传感器域适应结果极具说服力，尤其对未见传感器（如N→W）表现优异。
  3. 鲁棒性实验涵盖真实世界损坏，证明表示质量高。
