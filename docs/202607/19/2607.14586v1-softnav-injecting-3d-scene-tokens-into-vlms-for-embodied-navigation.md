## 摘要
在目标导向的具身导航中，智能体必须在未见环境中定位指定目标，这需要三维场景理解与导航推理协同工作。当前方法通过文本将三维场景信息传递给视觉语言模型（VLM），这在我们测试的配置中表明存在表征鸿沟；一项受控消融实验证实，直接嵌入级传输显著优于所评估的文本序列化格式。我们提出SoftNav，该方法通过轻量投影器将实体级三维连续表示（每个检测到的物体或前沿对应一个令牌）作为软令牌注入VLM的隐藏空间。在三维编码器和VLM冻结的情况下，仅需约1200个样本和约1700万可训练参数。在HM3D-OVON上，SoftNav在三个子集上分别实现了74.2%/68.3%/66.7%的成功率（SR），在成功率和路径长度加权成功率（SPL）上均超越了所有先前方法；同一导航策略可零样本迁移至GOAT-Bench（67.2% SR）、SG3D（47.2%语义成功率s-SR）以及真实世界机器人部署，无需重新训练或架构修改。将三维场景令牌直接注入VLM弥合了表征鸿沟，使得通过最少训练即可实现可迁移的导航。

## Abstract
In goal-directed embodied navigation, where an agent must locate a specified target in an unseen environment, 3D scene understanding and navigation reasoning must work in concert. Current approaches transmit 3D scene information to vision-language models (VLMs) through text, suggesting a representation gap in our tested configurations; a controlled ablation confirms that direct embedding-level transfer significantly outperforms the evaluated text serialization formats. We introduce SoftNav, which injects entity-level 3D continuous representations -- one token per detected object or frontier -- into a VLM's hidden space as soft tokens through a lightweight projector. With the 3D encoder and VLM frozen, only ~1,200 samples and ~17M trainable parameters are needed. On HM3D-OVON, SoftNav achieves 74.2%/68.3%/66.7% SR across three splits, surpassing all prior methods in both SR and SPL; the same navigation policy transfers zero-shot to GOAT-Bench (67.2% SR), SG3D (47.2% s-SR), and real-world robot deployment without retraining or architectural modification. Injecting 3D scene tokens directly into VLMs bridges the representation gap, enabling transferable navigation with minimal training.

- **研究背景**：目标导向的具身导航要求智能体在未见过的3D环境中定位指定目标（如“找到马桶”）。现有工作分为两类：模块化方法（如语义地图、3D策略）和基于VLM的方法。结合3D感知与VLM推理的现有方法普遍通过**文本序列化**（如坐标字符串、场景图文本）将3D场景信息传递给VLM。
- **核心问题**：文本序列化会产生“表征鸿沟”——连续、高维的3D特征（几何、语义、空间关系）无法被离散文本完整保留。该鸿沟尚未被量化，且是否可通过丰富文本或模型微调弥补未知。
- **整体含义**：本文首次**量化**了这一表征鸿沟，并提出SoftNav，通过将3D场景的连续嵌入直接注入VLM隐藏空间（软令牌注入）来弥合鸿沟，使得在极少训练数据（~1200样本）和极小参数量（~17M）下实现SOTA导航性能，并零样本迁移至多个基准和真实机器人。

## (2) 方法论

### 核心思想
- 使用预训练的3D场景编码器（PQ3D）提取每个物体和前沿的连续查询嵌入（768维），通过轻量MLP投影器映射到VLM的隐藏空间（2048维）作为“软令牌”。这些软令牌与视觉记忆令牌、文本指令令牌拼接后输入冻结的VLM（Qwen2.5-VL-3B）进行自回归预测，生成目标坐标，再snap到最近的前沿点。

### 关键技术细节
1. **3D场景编码**：采用PQ3D，从流式RGB-D观测中增量构建3D实例分割、维护物体库和检测前沿。对每个物体/前沿产生768维嵌入，编码了几何、语义和空间关系。
2. **视觉编码**：使用冻结的DINOv3从每帧图像提取patch特征，经压缩头（PatchMerger）降为15个令牌，并维护一个最近K=16帧的滚动缓冲区（共240个视觉令牌）。
3. **软令牌注入**：
   - **投影器**：两层MLP（768→1408→2048），中间使用GELU激活和LayerNorm。参数~4M。
   - **注入方式**：在文本模板中插入特殊占位符，在嵌入层将占位符替换为投影后的软令牌。输入序列顺序：视觉令牌 → 3D软令牌（前沿在前，物体在后）→ 文本令牌。
4. **VLM决策**：VLM生成形如“0.5, 3.2”的目标坐标字符串。将该坐标snap到最近的合法前沿点（距离比例限制r=2）。若前沿集为空则触发救援回退。
5. **训练策略**：
   - 冻结DINOv3、PQ3D、VLM主干。
   - 仅训练MLP投影器 + LoRA适配器（应用于注意力层和前馈层，r=8, α=32，~13M参数）。
   - 训练数据：从MTU3D系统成功轨迹中提取的~1200个监督样本（跨架构知识蒸馏）。
   - 损失函数：自回归交叉熵，以教师的前沿坐标字符串为标签。

### 算法流程（文字描述）
- 每步：① 压缩当前RGB帧获得视觉令牌，更新视觉记忆；② PQ3D从RGB-D+位姿提取物体/前沿嵌入；③ MLP投影器映射嵌入；④ 拼接视觉记忆、软令牌、文本指令；⑤ VLM生成目标坐标；⑥ 若前沿非空则snap，否则救援回退；⑦ 执行导航到选中前沿。
